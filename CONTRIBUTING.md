# Contributing to CDR

Thank you for your interest in contributing to the Conversational Demand Response project. We welcome contributions from the research community and industry practitioners.

## Getting Started

1. **Fork the repository** on GitHub
2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/YOUR-USERNAME/cdr.git
   cd cdr
   ```
3. **Create a branch** for your changes:
   ```bash
   git checkout -b feature/your-feature-name
   ```

## Types of Contributions

### Code
- New appliance agents or tool implementations
- Improvements to the ReAct orchestration loop
- Battery optimizer enhancements
- Dashboard UI improvements
- Additional LLM provider integrations

### Research
- Benchmark results with different LLM providers or models
- Extensions to multi-household or community-level DR
- Integration with real hardware or smart home platforms
- Comparative studies with rule-based or RL-based approaches

### Documentation
- Setup guides for different platforms
- API usage examples
- Architecture explanations

### Issues
- Bug reports with reproduction steps
- Feature requests with use-case descriptions

## Development Setup

Follow the [Setup Guide](docs/SETUP.md) to configure your environment.

## Submitting Changes

1. Commit your changes with a clear message
2. Push to your fork
3. Open a Pull Request against the `main` branch
4. Describe your changes and link any related issues
